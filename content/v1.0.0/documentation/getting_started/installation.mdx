# Installation Guide

Get Acme Platform up and running in your project in just a few minutes.

## Prerequisites

Before installing Acme CLI, make sure you have:

<CheckList>
  <CheckItem variant="do">
    Node.js 18 or higher installed
  </CheckItem>
  <CheckItem variant="do">
    Git repository with your project
  </CheckItem>
  <CheckItem variant="do">
    Package manager (npm, yarn, or pnpm)
  </CheckItem>
</CheckList>

## Installation Methods

### Method 1: NPM Package (Recommended)

Install Acme CLI globally or as a dev dependency:

```bash
# Global installation
npm install -g acme-cli

# Or as a dev dependency
npm install --save-dev acme-cli
```

### Method 2: Using npx

Run Acme CLI without installing:

```bash
npx acme-cli init
```

## Quick Setup

1. **Initialize your project**

```bash
acme init
```

This command will:
- Create an `acme.config.json` file
- Set up the basic project structure
- Install necessary dependencies

2. **Configure your project**

Edit the generated `acme.config.json`:

```json
{
  "name": "My Project",
  "description": "Project description",
  "environment": "development",
  "region": "us-east-1"
}
```

3. **Deploy your first application**

```bash
acme deploy
```

> [!TIP]
> Use acme deploy --watch to automatically redeploy when your code changes.

## Verification

To verify your installation is working correctly:

```bash
acme --version
```

You should see the version number displayed.

## Development Server

Start the development server to preview your application:

```bash
acme dev
```

Your application will be available at `http://localhost:3000`.

> [!WARNING]
> Make sure port 3000 is available, or specify a different port using --port flag.

## Troubleshooting

### Common Issues

<CheckList>
  <CheckItem variant="dont">
    **Port already in use**: Use `acme dev --port 3001` to specify a different port
  </CheckItem>
  <CheckItem variant="dont">
    **Permission errors**: Try running with `sudo` on macOS/Linux or as Administrator on Windows
  </CheckItem>
  <CheckItem variant="dont">
    **Node version issues**: Ensure you're using Node.js 18 or higher
  </CheckItem>
</CheckList>
